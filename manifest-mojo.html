opencms:manifest

Full name:

org.synyx:maven-opencms:1.2-SNAPSHOT:manifest

Description:

This Mojo creates a manifest.xml-File by delegating to ManifestBuilderTask created by Eurelis.

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: package.

Required Parameters

Name Type Since Description
exportversion String - The Export-Version (try 4 or 6) of the module.
opencmsversion String - The OpenCms-Version to create the module for

Optional Parameters

Name Type Since Description
author String - The Name of the Author of the Module. Defaults to Admin
authoremail String - The Email of the Author of the Module.
creator String - The creator of the module. Defaults to Admin
dateinstalled String - The Date this module was installed.
dependencies Properties - Dependencies of the Module as Properties.
distfolder String - This is the temporary folder where output is generated to. All mojos somehow interact using this folder: The createtree goal copies its data to there, the manifest goal creates its manifest there and the zipmodule goal zips this folder. WARNING: This folder gets deleted if it already exists when you execute the createtree goal. Defaults to ${project.basedir}/target/opencms
explorertypes String - Extra Explorertypes. Path to XML
exportpoints Properties - ExportPoints of the Module as Properties.
generateuuids Boolean - Flag wether to create UUIDs. Defaults to false
group String - The Group of the Module. Defaults to the Group of the Project
moduleDir String - Where the modules zip should be created to. Defaults to ${project.basedir}/target/
moduleclass String - The ActionClass of the Module.
moduledescription String - The Description of the Module. Defaults to the Description of the Project
name String - The Name of the Module. Defaults to the projects artifactid.
nicename String - The Nice-Name of the Module. Defaults to the Name of the Project
parameters Properties - Parameters of the Module as Properties.
project String - The Project of the Module. Defaults to Offline
projectBaseDir File - (no description)
resources List - Resources of the Module as List.
resourcetypes String - Extra Resourcetypes. Path to XML
targetLibPath String - If targetLibPath is given the package that is created during the package-phase of the lifecycle. The file that can be found unter ${project.build.directory}/${project.build.finalName}.jar will be copied to the given targetLibPath. Also there is an extra export-point added to the manifest for this directory.
userinstalled String - The User that has installed the Module. Defaults to Admin
version String - The Version of the Module. Defaults to the Version of the Project
